# SpecControl AI

AI-система проверки технической документации на соответствие нормативам

## Проблема

Инженеры, проектировщики и специалисты по нормоконтролю ежедневно вручную проверяют техническую документацию (проектные решения, спецификации, технические условия) на соответствие ГОСТам, СНиПам, СП и отраслевым стандартам. Это:

- **Медленно** — проверка одного документа может занимать часы и дни
- **Дорого** — требуется высококвалифицированный персонал с глубоким знанием нормативной базы
- **Ненадёжно** — человек пропускает несоответствия из-за усталости и объёма данных
- **Не масштабируется** — при росте объёма документации узкое место — эксперт

## Для кого

- Отделы нормоконтроля на предприятиях
- Проектные и инжиниринговые организации
- Строительные компании, работающие с проектной документацией
- Государственные экспертизы и надзорные органы

## Что делает PoC на демо

1. **Загрузка документа** — принимает PDF или DOCX с техническими параметрами (например, проект электроснабжения, спецификация материалов)
2. **Извлечение параметров** — агент извлекает ключевые технические параметры из документа (размеры, материалы, допуски, нагрузки, коэффициенты)
3. **Сопоставление с нормативами** — извлечённые параметры сверяются с базой нормативных требований (загруженные ГОСТы/СП)
4. **Выявление несоответствий** — система находит расхождения между фактическими значениями в документе и требованиями нормативов
5. **Формирование отчёта** — генерируется структурированный отчёт с перечнем найденных несоответствий, ссылками на конкретные пункты нормативов и рекомендациями

## Что НЕ делает PoC (out-of-scope)

- **Не заменяет эксперта** — система выступает как помощник, финальное решение остаётся за человеком
- **Не работает с чертежами и CAD-файлами** — только текстовые документы (PDF/DOCX)
- **Не покрывает все нормативы** — PoC работает с ограниченной базой из 3–5 выбранных нормативных документов
- **Не выполняет расчёты** — система сверяет заявленные значения, но не пересчитывает инженерные формулы
- **Не имеет production-уровня безопасности** — нет полноценной авторизации, аудита, шифрования хранилища
- **Не интегрируется с внешними системами** — нет подключения к САПР, СЭД или другим корпоративным системам
- **Не гарантирует SLA** — это демонстрационный прототип, не production-сервис
