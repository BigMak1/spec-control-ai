# Error Catalog

Каталог типовых ошибок проектной документации, выписанных из публичных источников (госэкспертизы, учебные материалы, СП/ПУЭ). Используется как **библиотека ошибок** для инжекции в baseline PDF при построении синтетического датасета Phase 0.

Каждая запись имеет стабильный `id`, по которому на неё ссылается `ground_truth.json.expected_violations[].note` в поле трассируемости.

## Категория 1 — Сечение кабелей vs ток нагрузки

Ключевые нормативы: **ПУЭ 7-е изд.** (гл. 1.3 «Выбор проводников по нагреву, экономической плотности тока и по условиям короны», 7.1 «Электрооборудование жилых, общественных, административных и бытовых зданий»), **ГОСТ Р 50571.5.52-2011** «Электропроводки».

| ID | Описание ошибки | Нарушаемый норматив | Инжектируемое значение (пример) |
|---|---|---|---|
| E01 | Сечение кабеля выбрано по номинальному току без учёта послеаварийных и ремонтных режимов | ПУЭ 1.3.1 | заменить 4 мм² → 2.5 мм² при расчётном токе 25 А |
| E02 | Не применены поправочные коэффициенты (температура среды, число параллельно проложенных кабелей, способ прокладки) при выборе сечения | ПУЭ 1.3.10–1.3.12 | в таблице указать расчётный ток без пересчёта для группы из 5 кабелей |
| E03 | Сечение кабеля не проверено по потере напряжения на длинной линии | ПУЭ 1.2.21, ГОСТ Р 50571.5.52 п. 525 | кабель 1.5 мм² на линии 80 м при токе 16 А |
| E04 | В здании применены кабели с алюминиевыми жилами вместо медных | ПУЭ 7.1.34 | указать марку АВВГ вместо ВВГнг в линии розеток |
| E05 | Сечение PE/нулевого защитного проводника меньше требуемого по отношению к фазному | ПУЭ 1.7.126, ГОСТ Р 50571.5.54 | PE = 1.5 мм² при фазном 4 мм² |
| E06 | Не указаны характеристики помещения (влажное, сырое, пыльное, пожароопасное) — невозможно обосновать выбор типа кабеля | ПУЭ 1.1.3, 7.1.36 | спецификация без указания класса помещения |

## Категория 2 — Воздухообмен в помещениях

Ключевые нормативы: **СП 60.13330.2020** «Отопление, вентиляция и кондиционирование воздуха» (разд. 7), **СанПиН 1.2.3685-21** «Гигиенические нормативы».

| ID | Описание ошибки | Нарушаемый норматив | Инжектируемое значение (пример) |
|---|---|---|---|
| V01 | Принятый воздухообмен по помещению не обоснован расчётом или нормативом (нет ссылки ни на СП, ни на СанПиН, ни на технологию) | СП 60.13330.2020 п. 7.2.1 | в таблице указан расход без графы «обоснование» |
| V02 | Таблица воздухообменов не содержит данных по балансу приточного и вытяжного воздуха | СП 60.13330.2020 пп. 7.5.1, 7.5.3, 7.5.4 | убрать колонку «вытяжка» или сделать приток > вытяжки в помещении общественного назначения |
| V03 | Для помещения без естественного проветривания с постоянным пребыванием людей не учтены требования к кратности воздухообмена | СП 60.13330.2020 п. 7.2.8 | кабинет без окон с кратностью 0.5 вместо ≥ 2 |
| V04 | Кратность воздухообмена меньше нормативной для функционального назначения помещения | СанПиН 1.2.3685-21 табл. 5.34 | столовая 20 м³/ч·чел вместо 60 м³/ч·чел |
| V05 | Не предусмотрены противопожарные клапаны на воздуховодах при пересечении противопожарных преград | СП 60.13330.2020 п. 7.11.13 | воздуховод через стену категории В1 без клапана |
| V06 | Неправильно рассчитан воздухообмен по вредным выделениям (углекислый газ, тепло, влага) | СП 60.13330.2020 п. 7.5 | расчёт только по кратности без учёта CO₂ в зале на 200 человек |

## Расширение каталога

Стартовый набор из 12 ошибок достаточен для Phase 0 (3–5 документов × 2–3 инжекции ≤ 15 уникальных ошибок). При расширении датасета приоритетные источники для добавления записей:

- [СПб госэкспертиза — электроснабжение](https://www.spbexp.ru/docs/tipovye-oshibki/sistema-elektrosnabzheniya/) — структурированный список; на момент составления каталога недоступен (таймаут)
- [СПб госэкспертиза — ОВиК](https://www.spbexp.ru/docs/tipovye-oshibki/otoplenie-ventilyatsiya-i-konditsionirovanie-vozdukha/)
- [Приморская госэкспертиза — общий список](https://primgosexpert.ru/tipovie_oshibki/) (пункты содержат в основном ошибки оформления ТУ и классификации помещений, а не расчётные)
- [Вологодская госэкспертиза](http://gosexpert35.ru/errors/)
- [CEUR — основные ошибки в проектной документации](https://ceur.ru/library/articles/stroitelnaja_jekspertiza/item196843/)
- [ПУЭ 7-е изд. (полный текст)](https://www.volt-spb.ru/media/docs/ntd/zazemlenie/pravila-ustroistva-elektroustanovok-pue-izdanie-sedmoe.pdf)
- [СП 60.13330.2020 (полный текст)](https://docs.cntd.ru/document/573697256)

## Правила использования при инжекции

1. Для каждого документа выбирать ошибки из разных категорий (электрика ИЛИ вентиляция в зависимости от baseline)
2. В одном документе не дублировать ошибки с одним и тем же `id`
3. При инжекции сохранять grammatical correctness документа — не ломать структуру таблицы, только подменить значение
4. Инжектированное значение фиксировать в `ground_truth.json.expected_violations[].actual`, а норматив — в `required` (цитата из норматива или ссылка на пункт)
5. В поле `note` обязательно указать `catalog_id: <id>` для трассируемости
